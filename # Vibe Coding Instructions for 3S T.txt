## Interactive Elements

### Animations
1. **Hero:** Gentle floating data particles moving from left (Cognos colors) to right (Power BI colors)
2. **Service Cards:** Staggered fade-in on scroll (100ms delay between each)
3. **Process Flow:** Animated dashed line connecting phases, drawing from left to right
4. **AI Section:** Pulsing glow effect around AI icon
5. **Hover States:** 
   - Cards: `transform: translateY(-8px)` + soft shadow
   - Buttons: Background gradient shift + slight scale

### Micro-interactions
- **CTAs:** Ripple effect on click
- **Navigation:** Underline slide-in on hover (cyan color)
- **Form Inputs:** Border glow (cyan) on focus
- **Stats Counters:** Count-up animation when scrolled into view

## Technical Implementation Notes

- **Framework:** React (with Tailwind CSS) for component reusability
- **Animations:** Framer Motion or GSAP for smooth interactions
- **Icons:** Lucide-react
- **Responsiveness:** Mobile-first, breakpoints at 640px, 1024px, 1280px
- **Performance:** Lazy-load images, optimize SVGs, use Intersection Observer for scroll animations

---

## Do's & Don'ts

‚úÖ **Do:**
- Use whitespace generously (BI audiences value clarity)
- Show process transparency (builds trust)
- Emphasize "AI-assisted" (not fully automated - implies oversight)
- Include specific technical terms (FM models, paginated reports) for credibility

‚ùå **Don't:**
- Overcomplicate navigation (single-page scroll or max 5 pages)
- Use stock photos of generic "business people"
- Make it look like a basic SaaS template (you're consultative, not commodity)
- Hide the technical complexity (your audience appreciates depth)
